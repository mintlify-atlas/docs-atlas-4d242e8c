## Overview

Views in SwiftFusePHP are template files that handle the presentation layer. They render HTML and display data passed from controllers. Views support both `.php` and `.html` file extensions.

## View Directory Structure

Views are organized in the `Vista/` directory, typically grouped by module:

```
Vista/
├── inicio/
│   ├── index_View.php
│   └── contact_View.php
├── user/
│   ├── profile_View.php
│   └── dashboard_View.php
└── error/
    ├── 404_View.php
    └── 403_View.php
```

<Note>
**Naming Convention:**
- File name: `{name}_View.{ext}` (e.g., `index_View.php`)
- The view name is lowercase
- Extensions can be `.php` or `.html`
</Note>

## Loading Views

Views are loaded from controllers using the `vista()` method:

<CodeGroup>
```php Controlador.php (Lines 55-69)
public function vista($vista, $parametro = [])
{
    $vista = lcfirst($vista);
    $extensions = array("html", "php");
    $isExist = false;
    foreach ($extensions as $extension) {
        $file = "Vista/" . $vista . "_View." . $extension;
        if (file_exists($file)) {
            $isExist = true;
            require_once $file;
            break;
        }
    }
    ($isExist ?: error(404));
}
```
</CodeGroup>

### How It Works

1. **Converts to lowercase:** View paths are converted to lowercase
2. **Checks extensions:** Tries `.html` first, then `.php`
3. **Requires file:** Uses `require_once` to include the view
4. **Error handling:** Returns 404 if view doesn't exist

## Passing Data to Views

Data is passed from controllers to views through the `$parametro` array:

<Tabs>
  <Tab title="Controller">
    ```php User_Controller.php
    public function profile($vista, $parametros)
    {
        $userId = $parametros[0];
        $user = $this->User->getById($userId);
        
        // Pass data to view
        $data = [
            'user' => $user,
            'title' => 'User Profile'
        ];
        
        $this->vista('user/profile', $data);
    }
    ```
  </Tab>
  
  <Tab title="View">
    ```php Vista/user/profile_View.php
    <!DOCTYPE html>
    <html>
    <head>
        <title><?php echo $title; ?></title>
    </head>
    <body>
        <h1><?php echo $user->name; ?></h1>
        <p>Email: <?php echo $user->email; ?></p>
        <p>Member since: <?php echo $user->created_at; ?></p>
    </body>
    </html>
    ```
  </Tab>
</Tabs>

<Note>
The `$parametro` array is extracted, making each key available as a variable in the view.
</Note>

## Default View Loading

When no specific method exists in the controller, the `cargaVista()` method is called:

<CodeGroup>
```php Controlador.php (Lines 23-29)
public function cargaVista($archivo, $data){
    if(count($data) == 0 ){
        $this->vista($this->folder . '/' . lcfirst($archivo), $data);
    }else{
        error(404);
    }
}
```
</CodeGroup>

**URL:** `http://example.com/inicio/contact`

**Result:** Loads `Vista/inicio/contact_View.php`

## View Examples

### Example 2: List View

<CodeGroup>
```php Controller: Product_Controller.php
public function list($vista, $parametros)
{
    $products = $this->Product->getAll();
    
    $data = [
        'products' => $products,
        'total' => count($products)
    ];
    
    $this->vista('product/list', $data);
}
```

```php View: Vista/product/list_View.php
<!DOCTYPE html>
<html>
<head>
    <title>Product List</title>
    <link rel="stylesheet" href="<?= RUTA_URL; ?>css/style.css">
</head>
<body>
    <h1>Products (<?php echo $total; ?>)</h1>
    
    <?php if ($total > 0): ?>
        <table>
            <thead>
                <tr>
                    <th>ID</th>
                    <th>Name</th>
                    <th>Price</th>
                    <th>Actions</th>
                </tr>
            </thead>
            <tbody>
                <?php foreach ($products as $product): ?>
                    <tr>
                        <td><?php echo $product['id']; ?></td>
                        <td><?php echo htmlspecialchars($product['name']); ?></td>
                        <td>$<?php echo number_format($product['price'], 2); ?></td>
                        <td>
                            <a href="<?= RUTA_URL; ?>product/view/<?= $product['id']; ?>">View</a>
                            <a href="<?= RUTA_URL; ?>product/edit/<?= $product['id']; ?>">Edit</a>
                        </td>
                    </tr>
                <?php endforeach; ?>
            </tbody>

    <?php else: ?>

    <?php endif; ?>
</body>
</html>
```

### Example 4: Detail View

<CodeGroup>
```php Controller: User_Controller.php
public function view($vista, $parametros)
{
    if (empty($parametros[0])) {
        error(404);
    }
    
    $userId = $parametros[0];
    $user = $this->User->getById($userId);
    
    if (!$user) {
        error(404);
    }
    
    $data = [
        'user' => $user,
        'orders' => $this->Order->getByUser($userId)
    ];
    
    $this->vista('user/detail', $data);
}
```

## Including Partial Views

You can include reusable view components:

<CodeGroup>
```php Main View
<!DOCTYPE html>
<html>
<head>
    <?php require_once RUTA_APP . "Includ/head.php"; ?>
</head>
<body>
    <?php require_once RUTA_APP . "Includ/header.php"; ?>
    
    <main>
        <!-- Your content here -->
    </main>
    
    <?php require_once RUTA_APP . "Includ/footer.php"; ?>
</body>
</html>
```
</CodeGroup>

<Tip>
**Create an Includ/ Directory**

Store reusable view components like headers, footers, and navigation in an `Includ/` directory.
</Tip>

## Working with JavaScript

Views can include JavaScript for dynamic behavior:

<CodeGroup>
```php View with AJAX
<!DOCTYPE html>
<html>
<head>
    <title>People List</title>
</head>
<body>
    <button id="btnListPeople" onclick="loadPeople()">Load People</button>
    <div id="peopleList"></div>
    
    <script>
    async function loadPeople() {
        const btn = document.getElementById('btnListPeople');
        btn.disabled = true;
        btn.textContent = 'Loading...';
        
        try {
            const response = await fetch('<?= RUTA_URL; ?>person/list');
            const result = await response.json();
            
            if (result.exito) {
                displayPeople(result.data);
            } else {
                alert(result.mensaje || 'Error loading data');
            }
        } catch (error) {
            alert('Network error');
        } finally {
            btn.disabled = false;
            btn.textContent = 'Load People';
        }
    }
    
    function displayPeople(people) {
        const container = document.getElementById('peopleList');
        container.innerHTML = people.map(person => `
            <div class="person">
                <h3>${person.name}</h3>
                <p>${person.email}</p>
            </div>
        `).join('');
    }
    </script>
</body>
</html>
```
</CodeGroup>

## Security in Views

<Warning>
**Always Escape User Data**

Use `htmlspecialchars()` to prevent XSS attacks:

```php
<!-- GOOD -->
<p><?php echo htmlspecialchars($user->name); ?></p>

<!-- BAD - XSS Vulnerability! -->
<p><?php echo $user->name; ?></p>
```
</Warning>

<CodeGroup>
```php XSS Prevention
<!-- Escape text content -->
<p><?php echo htmlspecialchars($userInput); ?></p>

<!-- Escape attributes -->
<input type="text" value="<?php echo htmlspecialchars($value); ?>">

<!-- For URLs -->
<a href="<?php echo htmlspecialchars($url); ?>">Link</a>

<!-- For JSON in JavaScript -->
<script>
const data = <?php echo json_encode($data); ?>;
</script>
```
</CodeGroup>

## View Helpers

Create helper functions for common tasks:

<CodeGroup>
```php helpers.php
<?php

function formatDate($date) {
    return date('F j, Y', strtotime($date));
}

function formatCurrency($amount) {
    return '$' . number_format($amount, 2);
}

function avatar($user) {
    if ($user->avatar) {
        return RUTA_URL . 'uploads/' . $user->avatar;
    }
    return RUTA_URL . 'images/default-avatar.png';
}

function url($path) {
    return RUTA_URL . ltrim($path, '/');
}
```

```php Using Helpers in Views
<p>Joined: <?php echo formatDate($user->created_at); ?></p>
<p>Balance: <?php echo formatCurrency($user->balance); ?></p>
<img src="<?php echo avatar($user); ?>" alt="Avatar">
<a href="<?php echo url('user/profile/' . $user->id); ?>">Profile</a>
```
</CodeGroup>

## Best Practices

<Tip>
**Separate Logic from Presentation**

Views should only display data, not perform business logic:

```php
<!-- GOOD -->
<?php foreach ($users as $user): ?>
    <p><?php echo $user->name; ?></p>
<?php endforeach; ?>

<!-- BAD -->
<?php
    $users = $User->getAll(); // Logic in view!
    foreach ($users as $user) {
        echo $user->name;
    }
?>
```
</Tip>

<Tip>
**Use Short Echo Tags**

For cleaner templates:

```php
<!-- Preferred -->
<title><?= $title; ?></title>

<!-- Also valid -->
<title><?php echo $title; ?></title>
```
</Tip>

<Tip>
**Organize by Module**

Group related views in subdirectories:

```
Vista/
├── user/
│   ├── list_View.php
│   ├── profile_View.php
│   └── settings_View.php
└── product/
    ├── list_View.php
    └── detail_View.php
```
</Tip>

<Warning>
**Avoid Complex Logic in Views**

Complex calculations and data manipulation belong in controllers or models:

```php
<!-- GOOD -->
<p>Total: <?= $total; ?></p>

<!-- BAD -->
<p>Total: <?= array_sum(array_column($orders, 'amount')) * 1.1; ?></p>
```
</Warning>
